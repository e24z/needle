# Example Traces

These traces are small, reviewable examples of Needle inside a Pi-shaped
session. They are not the benchmark table and should not be cited as quality
metrics. Start with `examples/README.md` for the plain-English overview.

`pi-sessions.jsonl` is generated by `scripts/generate-pi-session-traces.mjs`.
The generator drives the Pi extension harness directly: native tool output,
Needle pruning, status message, statusline state, and `/needle original`
recovery all land in one transcript-shaped record.

When an Apple Silicon machine has enough free memory, regenerate real-model
traces with:

```bash
PYTHONPATH=python \
NEEDLE_MODEL_DIR=/path/to/code-pruner \
NEEDLE_PYTHON=/path/to/needle/python/venv/bin/python \
node scripts/generate-pi-session-traces.mjs --backend real
```

For CI-friendly reproduction without MLX, use `--backend trace`. Those records
are useful for exercising the shape, but the committed examples should say so
if they use it.

The intended reading is:

- fixtures show the raw observation text;
- trace records show the user turn, tool call, output, status, original
  recovery, counters, and invariants;
- the verifier in `tests/test_example_traces.py` keeps the committed records in
  sync with their fixtures.
